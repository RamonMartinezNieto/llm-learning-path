Glosario de IA

Terminología canónica del workspace de aprendizaje. Actualizado conforme el usuario domina cada concepto.

Modelos y Arquitectura

LLM (Large Language Model)
Modelo de red neuronal entrenado sobre grandes volúmenes de texto para predecir la siguiente palabra (token). Capaz de generación, razonamiento y conversación.
Evitar: "IA generativa" como sinónimo exacto, "chatbot" para referirse al modelo.
Token
Unidad mínima de texto que procesa un LLM. Puede ser una palabra completa, una parte de palabra o un signo de puntuación. ~0.75 palabras en promedio en inglés.
Ventana de contexto (Context Window)
Número máximo de tokens que un modelo puede procesar en una sola inferencia, incluyendo prompt y respuesta. Lo que queda fuera de esta ventana, el modelo no lo "ve".
Evitar: "memoria del modelo" (confunde con fine-tuning).
Temperatura
Parámetro (0.0–2.0) que controla la aleatoriedad de la generación. Temperatura 0 → respuestas deterministas; temperatura alta → respuestas más variadas y creativas.
Embedding
Representación numérica densa (vector) de un texto en un espacio de alta dimensión, donde la proximidad entre vectores refleja similitud semántica.
Transformer
Arquitectura de red neuronal basada en mecanismo de atención, que es la base de todos los LLMs modernos (GPT, Claude, Gemini).
Alucinación (Hallucination)
Generación por parte del LLM de información plausible pero factualmente incorrecta o inventada, presentada con confianza.

Prompting

Prompt
Texto de entrada que se proporciona al LLM para obtener una respuesta. Incluye instrucciones, contexto, ejemplos y la pregunta o tarea.
Zero-shot
Técnica de prompting donde se pide al modelo realizar una tarea sin proporcionar ejemplos previos.
Few-shot
Técnica donde se incluyen 2–5 ejemplos de entrada/salida dentro del prompt para guiar el formato o razonamiento del modelo.
Chain-of-Thought (CoT)
Técnica que instruye al modelo a razonar paso a paso antes de dar una respuesta final. Mejora significativamente tareas de razonamiento complejo.
System prompt
Instrucciones de alto nivel que definen el rol, comportamiento y restricciones del modelo, separadas del mensaje del usuario.

APIs y Construcción

Inferencia
El proceso de ejecutar un modelo entrenado para generar una respuesta. Lo que ocurre cada vez que llamas a la API.
Tool use / Function calling
Capacidad de un LLM de invocar funciones externas predefinidas (buscar en web, ejecutar código, consultar BD) durante su razonamiento.
Structured output
Respuesta del modelo en formato estructurado garantizado (JSON, XML) según un schema predefinido.
Streaming
Entrega de la respuesta del modelo token a token en tiempo real, en lugar de esperar a que esté completa.
Prompt caching
Mecanismo que reutiliza el procesamiento de partes del prompt que no cambian entre llamadas, reduciendo latencia y coste.

Sistemas Agénticos

Agente (Agent)
Sistema donde un LLM actúa en un bucle de razonamiento–acción–observación para completar tareas complejas de forma autónoma.
ReAct
Patrón de agente que alterna entre Razonamiento (Reasoning) y Acción (Acting), usando los resultados de las acciones para continuar el razonamiento.
RAG (Retrieval-Augmented Generation)
Patrón que combina recuperación de documentos relevantes (búsqueda) con generación del LLM, inyectando el contexto recuperado en el prompt.
Orquestador
Componente que coordina y dirige múltiples agentes o herramientas hacia un objetivo común en un sistema multi-agente.

Producción

Fine-tuning
Proceso de continuar el entrenamiento de un modelo base con datos específicos del dominio para especializar su comportamiento.
Eval (Evaluación)
Conjunto de tests y métricas para medir sistemáticamente la calidad de las respuestas de un LLM en una tarea específica.
Prompt injection
Ataque donde entrada maliciosa del usuario intenta sobreescribir o ignorar las instrucciones del system prompt.
Latencia (TTFT)
Time To First Token: tiempo desde que se envía la petición hasta que llega el primer token de respuesta. Métrica clave de rendimiento en producción.
