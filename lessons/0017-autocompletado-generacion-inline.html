3.1 — Autocompletado y generación inline

Fase 3 Lección 17 de 55 · ⏱ ~12 min · IA en el Entorno de Desarrollo

Cómo funciona el autocompletado con IA

El autocompletado generativo — popularizado por GitHub Copilot — funciona mostrando texto fantasma (ghost text): una sugerencia semitransparente que aparece a la derecha del cursor mientras escribes. Presionas Tab para aceptarla, Esc para descartarla, o continúas escribiendo para ignorarla.

Bajo el capó, el modelo recibe como contexto el fichero activo, los ficheros abiertos en otras pestañas y, en herramientas más avanzadas, el repositorio completo. Con esa información predice el token más probable que continúa tu código.

  • Ghost text: sugerencia en línea, se acepta con Tab.
  • Generación inline: bloque de código generado desde un comentario o prompt en el propio editor.
  • Chat en el IDE: conversación separada; el modelo puede leer el contexto pero la respuesta no se inserta directamente en el cursor.

Autocompletado vs. generación: diferencias clave

El autocompletado reactivo completa lo que ya estás escribiendo. La generación proactiva parte de un comentario o una firma vacía y produce todo el cuerpo de una función. Aunque parecen iguales visualmente, el uso óptimo difiere:

  • Autocompletado: ideal para completar patrones repetitivos, imports, llamadas a métodos conocidos.
  • Generación inline: ideal para funciones de utilidad, transformaciones de datos, bloques de lógica bien acotada.
  • Edición inline (Ctrl+I / Cmd+I): modifica código existente según una instrucción textual dentro del editor.

La regla práctica: cuanto más mecánico y predecible sea el código, mayor será la fiabilidad de la sugerencia.

Cuándo confiar y cuándo verificar

Las sugerencias inline son más fiables en código que sigue patrones conocidos (CRUD, transformaciones de arrays, validaciones estándar). Requieren más escrutinio en:

  • Lógica de seguridad, autenticación y autorización.
  • Manejo de datos sensibles o cifrado.
  • Algoritmos con invariantes de negocio específicos.
  • Código que interactúa con APIs externas o sistemas de pago.

La mejor práctica fundamental: escribe primero el comentario o la firma de la función antes de dejar que el modelo genere el cuerpo. Esto actúa como especificación mínima y mejora drásticamente la relevancia de la sugerencia.

💡 "El código sugerido es una propuesta, no una respuesta. Siempre léelo antes de aceptarlo."
