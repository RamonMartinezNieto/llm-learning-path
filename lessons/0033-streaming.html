5.5 — Streaming de respuestas

Fase 5 · Lección 33 de 55 · ~12 min · Construcción con APIs de IA

Por defecto, las APIs de LLM generan la respuesta completa antes de enviártela. Con streaming, los tokens llegan a medida que se generan, lo que transforma radicalmente la percepción de velocidad por parte del usuario.

¿Por qué importa el streaming?

Imagina que un modelo tarda 8 segundos en generar una respuesta de 400 palabras. Sin streaming, el usuario ve una pantalla en blanco durante 8 segundos y luego todo el texto de golpe. Con streaming, empieza a ver texto en los primeros 300ms y la experiencia parece mucho más rápida —aunque el tiempo total sea idéntico.

Percepción vs. realidad El streaming mejora drásticamente la percepción de velocidad. Un modelo que tarda 5 segundos sin streaming parece mucho más lento que uno que empieza a responder en 0.3 segundos. El tiempo total es el mismo; la experiencia es completamente diferente.

Cómo funciona técnicamente

El streaming usa Server-Sent Events (SSE): el servidor mantiene la conexión HTTP abierta y envía eventos con pequeños fragmentos (delta chunks) de texto a medida que los genera. Tu cliente acumula esos deltas para construir la respuesta completa.

# Estructura de un evento SSE de Anthropic
event: content_block_delta
data: {"type": "content_block_delta", "index": 0, "delta": {"type": "text_delta", "text": "Hola"}}

event: content_block_delta  
data: {"type": "content_block_delta", "index": 0, "delta": {"type": "text_delta", "text": ", ¿cómo"}}

event: content_block_delta
data: {"type": "content_block_delta", "index": 0, "delta": {"type": "text_delta", "text": " puedo ayudarte?"}}

Streaming con Anthropic

La librería oficial de Anthropic ofrece un context manager stream() que gestiona la acumulación automáticamente:

import anthropic

client = anthropic.Anthropic()

# Opción 1: streaming simplificado con helper
with client.messages.stream(
    model="claude-opus-4-5",
    max_tokens=1024,
    messages=[{"role": "user", "content": "Explica la fotosíntesis en detalle."}]
) as stream:
    for fragmento in stream.text_stream:
        print(fragmento, end="", flush=True)  # Imprime token a token
    
    # Al salir del context manager, la respuesta completa está disponible
    mensaje_final = stream.get_final_message()
    print(f"\n\nTokens usados: {mensaje_final.usage.input_tokens}")
import anthropic

client = anthropic.Anthropic()

# Opción 2: control granular sobre los eventos
with client.messages.stream(
    model="claude-opus-4-5",
    max_tokens=1024,
    messages=[{"role": "user", "content": "Explica la fotosíntesis."}]
) as stream:
    texto_acumulado = ""
    
    for evento in stream:
        if hasattr(evento, 'delta') and hasattr(evento.delta, 'text'):
            fragmento = evento.delta.text
            texto_acumulado += fragmento
            # Aquí podrías enviar el fragmento a tu frontend via WebSocket
            print(fragmento, end="", flush=True)

Streaming con OpenAI

from openai import OpenAI

client = OpenAI()

stream = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "Explica la fotosíntesis en detalle."}],
    stream=True  # ← Activar streaming
)

texto_acumulado = ""
for chunk in stream:
    if chunk.choices[0].delta.content is not None:
        fragmento = chunk.choices[0].delta.content
        texto_acumulado += fragmento
        print(fragmento, end="", flush=True)

print()  # Nueva línea al final
print(f"Respuesta completa: {len(texto_acumulado)} caracteres")

Streaming en aplicaciones web con FastAPI

En una API web, usas StreamingResponse para reenviar los tokens al frontend:

from fastapi import FastAPI
from fastapi.responses import StreamingResponse
import anthropic

app = FastAPI()
client = anthropic.Anthropic()

@app.post("/chat/stream")
async def chat_stream(mensaje: str):
    def generar():
        with client.messages.stream(
            model="claude-opus-4-5",
            max_tokens=1024,
            messages=[{"role": "user", "content": mensaje}]
        ) as stream:
            for fragmento in stream.text_stream:
                # Formato SSE para el cliente
                yield f"data: {fragmento}\n\n"
        yield "data: [DONE]\n\n"
    
    return StreamingResponse(generar(), media_type="text/event-stream")

¿Cuándo NO usar streaming?

Situación¿Streaming?Razón
Chatbot conversacional✅ SíMejora mucho la experiencia del usuario
Structured outputs / JSON❌ NoNecesitas el JSON completo para parsearlo
Function calling / tool use❌ NoNecesitas la respuesta completa antes de ejecutar
Procesamiento en batch❌ NoNo hay usuario esperando, el throughput importa más
Generación de documentos⚠️ OpcionalDepende de si el usuario ve la generación en tiempo real
No uses streaming con structured outputs Si necesitas parsear el JSON de la respuesta, debes esperar a tener el texto completo. El streaming y el parsing de structured outputs son incompatibles a menos que implementes un parser de streaming JSON, lo cual es complejo y raramente vale la pena.
