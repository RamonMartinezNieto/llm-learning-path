4.5 — Terminales y CLI asistidos por IA

Fase 4 · Lección 27 de 55 · ~12 min · IA en el Ciclo DevOps

Herramientas de terminal asistidas por IA

El terminal es el entorno donde los desarrolladores pasan una fracción enorme de su tiempo, y es uno de los últimos espacios en integrar asistencia de IA de forma nativa. Esto está cambiando rápidamente con herramientas que convierten el lenguaje natural en comandos de shell precisos.

Las principales herramientas disponibles hoy:

  • GitHub Copilot en el terminal: integrado en gh CLI y en terminales compatibles como iTerm2 y Windows Terminal. Permite preguntar en lenguaje natural y obtener el comando correspondiente. Usa gh copilot suggest para sugerencias o gh copilot explain para entender un comando.
  • Warp: terminal nativo para macOS y Linux con IA integrada. Permite escribir "I want to find all files modified in the last 7 days larger than 1MB" y obtiene el comando find correcto.
  • llm (Simon Willison): herramienta CLI que permite interactuar con múltiples modelos de lenguaje directamente desde el terminal. Ideal para pipelines de texto, transformaciones de datos y scripting.
  • aider: agente de coding desde el terminal que edita archivos de código directamente usando instrucciones en lenguaje natural.

Casos de uso: de lenguaje natural a comandos precisos

El valor principal de la IA en el terminal es traducir intenciones expresadas en lenguaje natural a la sintaxis exacta de comandos complejos. Los comandos donde esto es más útil:

  • awk y sed: procesamiento de texto con sintaxis críptica que nadie recuerda de memoria. Ejemplo: "extrae la tercera columna de un CSV ignorando la primera línea de cabecera".
  • jq: transformación de JSON desde el terminal. La sintaxis de filtros de jq es poderosa pero difícil de escribir correctamente a la primera.
  • git con flags específicos: comandos como git log --oneline --graph --decorate --all --since="2 weeks ago" son difíciles de recordar pero la IA los genera al instante.
  • find con condiciones complejas: buscar archivos con múltiples criterios combinados (tipo, fecha, tamaño, permisos).
  • curl con headers y autenticación: llamadas a APIs REST con headers específicos, autenticación Bearer y cuerpo JSON.
# Ejemplo de prompt para la IA en terminal:
# "Encuentra todos los archivos .log en /var/log modificados
#  en las últimas 24 horas y comprímalos en un archivo tar.gz"

find /var/log -name "*.log" -mtime -1 | \
  tar -czf logs-$(date +%Y%m%d).tar.gz -T -
⚠️ Regla de oro del terminal con IA

Nunca ejecutes un comando generado por IA sin entender qué hace, especialmente si involucra rm, chmod, sudo o acceso a la red. Pide siempre a la IA que explique el comando antes de ejecutarlo.

💡 Ideal para la IA

Los comandos de una sola línea con pipes (|), sed, awk y jq son perfectos para la IA — son difíciles de recordar pero siguen patrones precisos.

Uso avanzado: llm en pipelines de texto

La herramienta llm de Simon Willison permite integrar modelos de lenguaje en pipelines de Unix, tratando la IA como un filtro de texto más:

# Resumir el output de un comando
git log --oneline -20 | llm "Resume estos commits en un párrafo"

# Analizar logs de error
tail -n 100 app.log | grep ERROR | llm "¿Cuál parece ser el problema principal?"

# Transformar datos
cat data.csv | llm "Convierte esto a JSON con los campos name y value"

Este paradigma convierte la IA en una herramienta de línea de comandos más, componible con grep, cat, pipe y el resto del ecosistema Unix.
