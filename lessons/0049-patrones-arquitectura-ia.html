7.6 — Patrones de arquitectura para sistemas IA robustos

Lección 49 de 55 · ~16 min · Fase 7: Sistemas de IA en Producción

A medida que una aplicación LLM crece y madura, emergen necesidades que requieren patrones arquitectónicos específicos. Estos patrones, heredados del diseño de sistemas distribuidos, se adaptan a las peculiaridades de los LLMs: latencia alta, fallos no binarios, y coste por token.

Patrón 1: Prompt Gateway

Un gateway centralizado que intercepta todas las llamadas al LLM antes de que lleguen a la API del proveedor. Responsabilidades: routing entre modelos, logging centralizado, control de costes, rate limiting, prompt versioning.

User Request → Prompt Gateway → [logging, routing, rate limit] → LLM API
                                      ↓
                              Metrics & Logs Store

Cuándo añadirlo: Cuando tienes múltiples features usando LLMs y necesitas visibilidad y control centralizado.

Patrón 2: Fallback Chain

Intentar con el modelo principal y, si falla o excede el timeout, usar automáticamente un modelo de backup más fiable o más barato.

def call_with_fallback(prompt):
    try:
        return call_model("claude-sonnet-3.5", prompt, timeout=10)
    except (Timeout, RateLimitError):
        return call_model("claude-haiku-3.5", prompt, timeout=20)
    except Exception:
        return fallback_response("Lo siento, el servicio no está disponible.")

Patrón 3: Circuit Breaker

Si el proveedor de LLM está fallando repetidamente, el circuit breaker "abre" el circuito y deja de llamar al proveedor por un periodo, evitando cascadas de errores y timeouts innecesarios. Tras el cooldown, prueba de nuevo.

Patrón 4: Human-in-the-loop

Checkpoints donde el agente pausa y espera confirmación humana antes de ejecutar acciones de alto impacto o irreversibles. El "human-in-the-loop" no es un anti-patrón — es una característica de seguridad deliberada.

Patrón 5: Async Processing

Para tareas que no son tiempo real (análisis de documentos, generación de informes, procesamiento de emails), encolar las llamadas al LLM en una queue (SQS, RabbitMQ) y procesarlas en background. Ventajas: no bloquea el usuario, permite retry automático, facilita el batching para descuentos de precio.

Cuándo añadir cada patrón

PatrónSeñal para añadirlo
Prompt GatewayMúltiples features, necesitas logging centralizado
Fallback ChainSLA de disponibilidad > 99%, usuarios en producción
Circuit BreakerObservas cascadas de errores cuando el proveedor falla
Human-in-the-loopEl agente ejecuta acciones irreversibles o de alto impacto
Async ProcessingLas tareas tardan > 10s y no necesitan respuesta inmediata
Regla de simplicidad Añadir todos estos patrones desde el día 1 es over-engineering. Empieza con la arquitectura más simple posible y añade robustez en respuesta a problemas reales.
