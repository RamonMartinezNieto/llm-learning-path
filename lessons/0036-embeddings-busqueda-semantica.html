5.8 — Embeddings y búsqueda semántica

Fase 5 · Lección 36 de 55 · ~16 min · Construcción con APIs de IA

Los embeddings son la tecnología que hace posible el RAG, la búsqueda semántica y muchos otros patrones de IA. Entender qué son y cómo usarlos te abre un abanico de capacidades que van mucho más allá de los chatbots.

¿Qué es un embedding?

Un embedding es una representación numérica densa de un texto como un vector de cientos o miles de dimensiones. La clave es que textos con significado similar tienen vectores cercanos en ese espacio multidimensional, independientemente de las palabras exactas usadas.

Ejemplo intuitivo: "perro", "can", "animal doméstico de cuatro patas" y "dog" (en inglés) producen embeddings muy cercanos entre sí, porque un buen modelo de embeddings captura el significado, no las palabras.

Concepto clave Los embeddings son el puente entre el lenguaje natural y las matemáticas. Permiten hacer búsquedas por significado, no por palabras clave. "¿Cuánto cuesta?" y "precio del producto" serán vecinos en el espacio de embeddings.

Cómo generar embeddings

Usas un modelo de embeddings dedicado (diferente al modelo de chat) para convertir texto en vectores:

from openai import OpenAI
import numpy as np

client = OpenAI()

def generar_embedding(texto: str) -> list[float]:
    """Convierte texto en un vector de 1536 dimensiones."""
    respuesta = client.embeddings.create(
        model="text-embedding-3-small",  # 1536 dimensiones, muy eficiente
        input=texto
    )
    return respuesta.data[0].embedding

# Generar embeddings
emb_perro = generar_embedding("perro")
emb_gato = generar_embedding("gato")
emb_can = generar_embedding("can")      # Sinónimo de perro
emb_avion = generar_embedding("avión")

print(f"Dimensiones del vector: {len(emb_perro)}")  # → 1536

Similitud coseno: la métrica de distancia

Para medir qué tan similares son dos embeddings, usamos la similitud coseno: mide el ángulo entre dos vectores. Valor 1.0 = idénticos, 0.0 = sin relación, -1.0 = opuestos.

import numpy as np
from openai import OpenAI

client = OpenAI()

def similitud_coseno(vec1: list, vec2: list) -> float:
    """Calcula similitud coseno entre dos vectores. Rango: -1 a 1."""
    a = np.array(vec1)
    b = np.array(vec2)
    return float(np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b)))

def comparar_textos(texto1: str, texto2: str) -> float:
    emb1 = client.embeddings.create(model="text-embedding-3-small", input=texto1).data[0].embedding
    emb2 = client.embeddings.create(model="text-embedding-3-small", input=texto2).data[0].embedding
    return similitud_coseno(emb1, emb2)

# Comparaciones
print(f"perro vs can:     {comparar_textos('perro', 'can'):.3f}")        # → ~0.85 (muy similares)
print(f"perro vs gato:    {comparar_textos('perro', 'gato'):.3f}")       # → ~0.70 (relacionados)
print(f"perro vs avión:   {comparar_textos('perro', 'avión'):.3f}")      # → ~0.25 (no relacionados)
print(f"precio vs coste:  {comparar_textos('precio', 'coste'):.3f}")     # → ~0.88 (muy similares)

Ejemplo completo: generar, almacenar y consultar

from openai import OpenAI
import chromadb
import numpy as np

openai_client = OpenAI()
chroma_client = chromadb.Client()
coleccion = chroma_client.create_collection(
    name="productos",
    metadata={"hnsw:space": "cosine"}  # Usar similitud coseno
)

# Catálogo de productos
productos = [
    {"id": "p1", "texto": "Zapatos de running ligeros para asfalto, talla 42"},
    {"id": "p2", "texto": "Zapatillas de trail running para montaña con suela grip"},
    {"id": "p3", "texto": "Camiseta técnica de secado rápido para ciclismo"},
    {"id": "p4", "texto": "Bicicleta de carretera con cambios Shimano 105"},
    {"id": "p5", "texto": "Mochila de hidratación 10L para trail running"},
]

# INDEXAR: Generar embeddings y almacenar
print("Indexando productos...")
embeddings = []
for producto in productos:
    emb = openai_client.embeddings.create(
        model="text-embedding-3-small",
        input=producto["texto"]
    ).data[0].embedding
    embeddings.append(emb)

coleccion.add(
    ids=[p["id"] for p in productos],
    documents=[p["texto"] for p in productos],
    embeddings=embeddings
)
print(f"✓ {len(productos)} productos indexados")

# BUSCAR: Consulta semántica
def buscar_productos(consulta: str, n: int = 3):
    # Generar embedding de la consulta
    emb_consulta = openai_client.embeddings.create(
        model="text-embedding-3-small",
        input=consulta
    ).data[0].embedding
    
    # Buscar en Chroma
    resultados = coleccion.query(
        query_embeddings=[emb_consulta],
        n_results=n,
        include=["documents", "distances"]
    )
    
    print(f"\nConsulta: '{consulta}'")
    print("Resultados más relevantes:")
    for doc, dist in zip(resultados["documents"][0], resultados["distances"][0]):
        similitud = 1 - dist  # Chroma devuelve distancia, convertir a similitud
        print(f"  [{similitud:.2f}] {doc}")

buscar_productos("calzado para correr en ciudad")
# → Encontrará zapatos de running aunque no use las palabras exactas

buscar_productos("bolsa de agua para deporte en montaña")
# → Encontrará la mochila de hidratación

Casos de uso más allá de RAG

  • Sistemas de recomendación: "Usuarios que compraron X también les gustó Y" basado en similitud de embeddings de productos/contenidos
  • Detección de duplicados: encontrar tickets de soporte, preguntas FAQ o documentos con el mismo significado
  • Clustering semántico: agrupar automáticamente feedback de clientes por tema sin categorías predefinidas
  • Clasificación zero-shot: clasificar texto en categorías nuevas sin ejemplos de entrenamiento
  • Búsqueda de código: encontrar funciones similares en un repositorio grande por lo que hacen, no por su nombre
El modelo de embeddings debe ser consistente Siempre usa el mismo modelo para indexar y para hacer las consultas. Si cambias el modelo, todos los embeddings almacenados quedan invalidados y debes reindexar. Dos modelos distintos producen espacios vectoriales incompatibles.
