6.1 — Arquitectura ReAct: Reasoning + Acting

Lección 37 de 55 · ~16 min · Fase 6: Código Agéntico

El patrón ReAct (Reasoning + Acting), propuesto por Yao et al. en 2022, es el fundamento de casi todos los agentes de IA modernos. La idea central es sencilla pero poderosa: en lugar de que el modelo solo razone (como en Chain-of-Thought) o solo ejecute acciones, el agente alterna de forma cíclica entre razonamiento explícito y ejecución de acciones, observando el resultado de cada acción antes de decidir el siguiente paso.

El bucle Thought → Action → Observation

El bucle ReAct se compone de tres tipos de pasos que se repiten hasta completar la tarea:

  • Thought (Razonamiento): El modelo reflexiona sobre el estado actual, lo que sabe y lo que necesita hacer a continuación.
  • Action (Acción): El modelo invoca una herramienta concreta — buscar en la web, ejecutar código, leer un archivo, llamar a una API.
  • Observation (Observación): El resultado de la acción se devuelve al modelo como nueva información que alimenta el siguiente ciclo de razonamiento.

Este ciclo permite que el agente integre información externa real en su razonamiento, corrigiendo su plan a medida que obtiene nuevos datos del mundo.

Ejemplo de traza ReAct

Pregunta: "¿Cuántos días lleva activo el proyecto Apache Kafka desde su primera release pública?"

Thought: Necesito saber la fecha de la primera release pública de Apache Kafka.
Action: search_web("Apache Kafka first public release date")
Observation: Apache Kafka fue liberado como open source en enero de 2011.

Thought: Ahora calculo cuántos días han pasado desde enero de 2011 hasta hoy.
Action: calculate_days("2011-01-01", "2025-01-15")
Observation: Han pasado aproximadamente 5,128 días.

Thought: Tengo todos los datos necesarios para responder.
Final Answer: Apache Kafka ha estado activo durante aproximadamente 5,128 días
desde su primera release pública en enero de 2011.

Sin ReAct, el modelo intentaría responder solo con su conocimiento de entrenamiento — posiblemente desactualizado. Con ReAct, puede obtener datos reales y realizar cálculos precisos en tiempo de inferencia.

ReAct vs. Chain-of-Thought puro

CaracterísticaChain-of-Thought (CoT)ReAct
Acceso a información externaNoSí, mediante herramientas
Corrección en tiempo realNoSí, tras cada observación
TransparenciaPasos de pensamiento visiblesPensamiento + acciones visibles
Casos de uso idealesRazonamiento con conocimiento del modeloTareas que requieren datos externos o actuales

Los experimentos del paper original mostraron que ReAct supera significativamente a CoT puro en tareas de búsqueda de información y resolución de problemas de múltiples pasos, precisamente porque puede verificar y actualizar su razonamiento con datos reales en cada paso.

Diferencia con el uso básico de herramientas

El function calling básico permite que un modelo llame a una función cuando lo necesita, pero sin un bucle de razonamiento explícito ni la capacidad de iterar basándose en observaciones. ReAct va más allá: el modelo escribe su razonamiento antes de cada acción, haciendo el proceso transparente, depurable y más efectivo para tareas de múltiples pasos. Es la diferencia entre "el modelo llama a una función" y "el modelo piensa, actúa, observa y actualiza su plan".

Concepto clave El patrón ReAct es la base de casi todos los agentes modernos. Comprender este bucle te permite diseñar y depurar sistemas agénticos.
