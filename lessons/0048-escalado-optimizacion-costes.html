7.5 — Escalado y optimización de costes en producción

Lección 48 de 55 · ~14 min · Fase 7: Sistemas de IA en Producción

El coste de las APIs de LLM puede escalar rápidamente si no se gestiona activamente. Aplicaciones que funcionan bien en desarrollo con 100 usuarios pueden resultar económicamente inviables con 10.000 si no se implementan estrategias de optimización. Entender los componentes del coste y las estrategias disponibles es esencial para escalar.

Componentes del coste

El coste básico de cada llamada es: tokens_entrada × precio_entrada + tokens_salida × precio_salida

Los tokens de salida son 3-5x más caros que los de entrada. Una respuesta larga puede costar mucho más que el prompt que la generó.

Estrategias de optimización

1. Model routing (mayor impacto): Clasificar las peticiones entrantes y usar el modelo más económico que pueda resolver cada tipo de tarea. Una petición de clasificación simple no necesita GPT-4o.

def route_request(user_query):
    complexity = classify_complexity(user_query)  # LLM barato
    if complexity == "simple":
        return call_model("gpt-4o-mini", user_query)
    else:
        return call_model("gpt-4o", user_query)

2. Prompt compression: Eliminar palabras innecesarias del system prompt. Un prompt de 2000 tokens se puede reducir a 800 sin perder comportamiento significativo.

3. Output length limits: Establecer max_tokens explícito. Sin límite, el modelo puede generar respuestas mucho más largas de lo necesario.

4. Prompt caching: Anthropic y OpenAI ofrecen caching de los prefijos del prompt. Si el system prompt es siempre el mismo, se cobra solo la primera vez.

5. Application-level cache: Para queries frecuentes y predecibles, cachear las respuestas en Redis o similar. Un FAQ respondido una vez puede servirse 1000 veces sin llamadas a la API.

6. Batching: Para procesamiento en batch (no tiempo real), agrupar múltiples requests y procesarlos con precios batch (50% descuento en Anthropic y OpenAI).

Advertencia sobre el contexto El coste de tokens puede escalar de forma no lineal cuando añades contexto. Un contexto de 10K tokens cuesta 5x más que uno de 2K tokens, pero no necesariamente da respuestas 5x mejores.
