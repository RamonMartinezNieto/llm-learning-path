5.6 — Gestión de conversaciones y memoria en aplicaciones

Fase 5 · Lección 34 de 55 · ~15 min · Construcción con APIs de IA

Los LLMs son stateless: cada llamada a la API es completamente independiente. No hay "sesión" en el servidor, no hay memoria entre llamadas. Cuando un chatbot parece recordar lo que dijiste antes, es porque tu código se lo está recordando explícitamente.

Principio fundamental La "memoria" de un chatbot no la gestiona el modelo — la gestiona TU CÓDIGO. Eres responsable de qué historial incluyes en cada llamada a la API.

Patrón 1: Historial completo en contexto

El más simple: acumulas todos los mensajes y los envías en cada llamada. Funciona perfectamente para conversaciones cortas.

import anthropic

client = anthropic.Anthropic()

class ChatSimple:
    def __init__(self):
        self.historial = []
    
    def enviar(self, mensaje_usuario: str) -> str:
        # Añadir mensaje del usuario al historial
        self.historial.append({
            "role": "user",
            "content": mensaje_usuario
        })
        
        # Llamar con TODO el historial
        respuesta = client.messages.create(
            model="claude-opus-4-5",
            max_tokens=1024,
            messages=self.historial
        )
        
        texto = respuesta.content[0].text
        
        # Guardar respuesta del asistente en el historial
        self.historial.append({
            "role": "assistant",
            "content": texto
        })
        
        return texto

chat = ChatSimple()
print(chat.enviar("Me llamo Carlos y trabajo como médico."))
print(chat.enviar("¿Qué hago profesionalmente?"))  # Recordará que eres médico

Problema: para conversaciones largas, el historial crece indefinidamente, aumentando el coste y eventualmente superando el límite de la ventana de contexto del modelo.

Patrón 2: Ventana deslizante (sliding window)

Mantén solo los últimos N mensajes. Simple y predecible, pero puede "olvidar" información importante de antes:

import anthropic
from collections import deque

client = anthropic.Anthropic()

class ChatVentanaDeslizante:
    def __init__(self, max_mensajes: int = 20):
        # deque con tamaño máximo — descarta automáticamente los más antiguos
        self.historial = deque(maxlen=max_mensajes)
        self.system = "Eres un asistente útil."
    
    def enviar(self, mensaje_usuario: str) -> str:
        self.historial.append({"role": "user", "content": mensaje_usuario})
        
        respuesta = client.messages.create(
            model="claude-opus-4-5",
            max_tokens=1024,
            system=self.system,
            messages=list(self.historial)  # Convierte deque a lista
        )
        
        texto = respuesta.content[0].text
        self.historial.append({"role": "assistant", "content": texto})
        return texto

chat = ChatVentanaDeslizante(max_mensajes=10)  # Solo últimos 10 mensajes

Patrón 3: Memoria por resumen (summarization)

Cuando el historial supera un umbral, pides al modelo que lo comprima en un resumen. El resumen reemplaza los mensajes antiguos, preservando la información esencial:

import anthropic

client = anthropic.Anthropic()

class ChatConResumen:
    def __init__(self, umbral_tokens: int = 2000):
        self.historial = []
        self.resumen = ""  # Resumen de la conversación pasada
        self.umbral_tokens = umbral_tokens
    
    def _contar_tokens_aprox(self) -> int:
        # Aproximación: 1 token ≈ 4 caracteres en español
        texto_total = " ".join(m["content"] for m in self.historial)
        return len(texto_total) // 4
    
    def _resumir_historial(self):
        """Comprime el historial antiguo en un resumen."""
        historial_texto = "\n".join(
            f"{m['role'].upper()}: {m['content']}" 
            for m in self.historial[:-4]  # Mantén los últimos 4 mensajes intactos
        )
        
        respuesta_resumen = client.messages.create(
            model="claude-opus-4-5",
            max_tokens=512,
            messages=[{
                "role": "user",
                "content": f"Resume esta conversación en 3-5 frases, preservando hechos clave:\n\n{historial_texto}"
            }]
        )
        
        nuevo_resumen = respuesta_resumen.content[0].text
        self.resumen = nuevo_resumen
        self.historial = self.historial[-4:]  # Mantén solo los últimos 4 mensajes
    
    def enviar(self, mensaje_usuario: str) -> str:
        if self._contar_tokens_aprox() > self.umbral_tokens:
            self._resumir_historial()
        
        self.historial.append({"role": "user", "content": mensaje_usuario})
        
        # Incluir el resumen como contexto en el system prompt
        system = "Eres un asistente útil."
        if self.resumen:
            system += f"\n\nResumen de la conversación anterior:\n{self.resumen}"
        
        respuesta = client.messages.create(
            model="claude-opus-4-5",
            max_tokens=1024,
            system=system,
            messages=self.historial
        )
        
        texto = respuesta.content[0].text
        self.historial.append({"role": "assistant", "content": texto})
        return texto

Patrón 4: Memoria externa (vector DB)

Para conversaciones muy largas o cuando necesitas recuperar información específica, puedes almacenar los mensajes en una base de datos vectorial y recuperar solo los fragmentos relevantes para cada nueva pregunta. Este es el patrón RAG aplicado a la memoria conversacional.

# Pseudocódigo del patrón de memoria semántica
# (Se desarrolla en detalle en las lecciones 5.7 y 5.8)

class ChatMemoriaSemantica:
    def __init__(self):
        self.db_vectorial = ChromaDB()  # o Pinecone, pgvector...
        self.historial_reciente = []    # Últimos N mensajes
    
    def enviar(self, mensaje_usuario: str) -> str:
        # 1. Buscar memorias relevantes para este mensaje
        memorias = self.db_vectorial.buscar_similares(
            query=mensaje_usuario, 
            n_resultados=5
        )
        
        # 2. Construir contexto con memorias + historial reciente
        contexto_memoria = "\n".join(memorias)
        
        # 3. Llamar al modelo con contexto enriquecido
        respuesta = llamar_api(
            system=f"Contexto relevante de conversaciones pasadas:\n{contexto_memoria}",
            mensajes=self.historial_reciente + [{"role": "user", "content": mensaje_usuario}]
        )
        
        # 4. Guardar nuevo intercambio en la DB vectorial
        self.db_vectorial.guardar(f"Usuario: {mensaje_usuario}\nAsistente: {respuesta}")
        
        return respuesta
