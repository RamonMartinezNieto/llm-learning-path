8.5 — IA para análisis de datos y visualización

Lección 54 de 55 · ~14 min · Fase 8: IA Aplicada a Otras Áreas

Antes, para analizar un CSV necesitabas saber pandas, matplotlib o al menos Excel avanzado. Con el code interpreter de ChatGPT y las capacidades de análisis de datos de Claude, cualquier persona puede convertir una pregunta en lenguaje natural en código Python ejecutable con visualizaciones en cuestión de segundos. Este es uno de los casos de uso de mayor ROI inmediato de la IA.

Code Interpreter: el analista bajo demanda

El code interpreter (disponible en ChatGPT Advanced Data Analysis y Claude) funciona así:

  1. Subes un archivo CSV, Excel o similar
  2. Describes en lenguaje natural qué quieres ver o analizar
  3. El LLM genera y ejecuta código Python (pandas, matplotlib, seaborn) en un sandbox
  4. Ves el resultado y la gráfica directamente en el chat
  5. Iteras con más preguntas o refinamientos

Ejemplo de workflow

# Tú: [subes ventas_2024.csv]
"Muéstrame las ventas por mes en un gráfico de barras.
Resalta los meses con crecimiento mayor al 20% respecto
al mes anterior."

# El LLM genera y ejecuta:
import pandas as pd
import matplotlib.pyplot as plt

df = pd.read_csv('ventas_2024.csv')
df['crecimiento'] = df['ventas'].pct_change() * 100
# ... genera el gráfico con highlighting condicional

Cuándo usar IA vs. herramientas BI dedicadas

Usa IA (code interpreter) cuando: análisis ad hoc, exploración de un dataset nuevo, presentaciones rápidas, no tienes acceso a BI tools, o el análisis es complejo y requiere lógica custom.

Usa herramientas BI (Tableau, Power BI, Metabase) cuando: dashboards recurrentes que se actualizan automáticamente, análisis en tiempo real sobre bases de datos en producción, o múltiples usuarios necesitan acceso centralizado a los mismos reportes.

Limitaciones importantes

  • Privacidad de datos: Los datos que subes al code interpreter van a los servidores del proveedor (OpenAI, Anthropic). Nunca subas datos confidenciales, PII, o información sujeta a NDA.
  • El modelo puede hacer suposiciones: Si el CSV tiene campos ambiguos, el modelo puede interpretarlos de forma incorrecta. Siempre revisa el código generado antes de confiar en el resultado.
  • Sesgo de confirmación: Si pides "demuestra que X está relacionado con Y", el modelo puede diseñar análisis que confirmen esa hipótesis. Formula preguntas abiertas.
