4.3 — IA para seguridad: detección de vulnerabilidades

Fase 4 · Lección 25 de 55 · ~14 min · IA en el Ciclo DevOps

Qué detecta bien la IA en un análisis de seguridad

La seguridad del software es un dominio donde la IA aporta valor real porque muchas vulnerabilidades tienen patrones reconocibles que la IA ha visto miles de veces en su entrenamiento. No es magia: es reconocimiento de patrones a gran escala.

Vulnerabilidades donde la IA destaca especialmente:

  • SQL Injection: queries construidas con concatenación de strings en lugar de parámetros preparados.
  • Cross-Site Scripting (XSS): datos de usuario insertados en el DOM sin sanitizar.
  • Hardcoded secrets: API keys, contraseñas y tokens escritos directamente en el código fuente.
  • Dependencias inseguras: identificar paquetes con CVEs conocidos o versiones muy desactualizadas.
  • Insecure Direct Object References (IDOR): endpoints que no verifican autorización antes de acceder a recursos.
  • Uso inseguro de criptografía: MD5 para passwords, IV fijo en cifrado, etc.
  • Path traversal: uso de input de usuario en operaciones de sistema de archivos sin sanitizar.

Plantilla de prompt para revisión de seguridad:

Analiza este código en busca de vulnerabilidades de seguridad.
Prioriza según el OWASP Top 10. Para cada vulnerabilidad encontrada:
1. Nombra la categoría OWASP
2. Señala la línea exacta
3. Explica por qué es peligrosa
4. Proporciona el código corregido

Herramientas: IA + SAST en la práctica

La IA no opera en aislamiento — se integra con herramientas de análisis estático (SAST) para crear pipelines de seguridad potentes:

  • GitHub Advanced Security: incluye CodeQL para análisis semántico del código y Secret Scanning automático. Copilot puede explicar los hallazgos y sugerir fixes.
  • Semgrep: motor de análisis estático muy configurable. Con IA puedes describir un patrón inseguro en lenguaje natural y Semgrep lo convierte en una regla.
  • Snyk: especializado en dependencias y containers, con sugerencias de remediación generadas por IA.
  • Copilot Autofix: cuando CodeQL detecta una vulnerabilidad, Copilot puede generar automáticamente el PR con el fix.
⚠️ Limitación importante

La IA no reemplaza un pentest ni un SAST configurado específicamente para tu stack. Es una capa adicional, no la definitiva. Las vulnerabilidades de lógica de negocio y las pruebas dinámicas requieren herramientas y expertos especializados.

Limitaciones críticas del análisis de seguridad con IA

Para usar la IA en seguridad de forma responsable, es clave entender dónde falla:

  • Vulnerabilidades de lógica de negocio: si el sistema permite comprar productos a precio negativo porque la lógica de descuentos es incorrecta, la IA no lo detectará — no conoce las reglas de negocio.
  • Vulnerabilidades de runtime: race conditions, problemas de concurrencia y vulnerabilidades que solo aparecen con datos o tráfico real.
  • Ataques de cadena de suministro: un paquete malicioso que tiene código aparentemente legítimo.
  • Falsos negativos por contexto: el código puede parecer seguro en aislamiento pero ser vulnerable en el contexto del sistema completo.
💡 Estrategia óptima

Para máximo impacto: ejecuta un SAST automático en cada PR y usa IA para revisar código nuevo de autenticación, autorización y manejo de datos sensibles. Combina ambas capas.
