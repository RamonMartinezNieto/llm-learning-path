Evaluación y refinamiento iterativo de prompts

Fase 1 · Lección 10 de 55 · ~14 min · Prompt Engineering

El bucle de refinamiento

Un prompt rara vez funciona a la primera para producción. El refinamiento iterativo es el proceso real de prompt engineering — no es arte, es ingeniería con feedback.

1

Definir criterio de éxito

¿Qué hace que una respuesta sea buena? Define esto antes de escribir el prompt, no después. Ej: "contiene los 3 puntos clave, en menos de 150 palabras, sin jerga técnica".

2

Crear un conjunto de evaluación

5–20 inputs representativos que cubran casos normales y edge cases. El prompt debe funcionar en todos ellos, no solo en el ejemplo que usaste para diseñarlo.

3

Ejecutar y evaluar

Corre el prompt en todos los inputs. Anota en qué falla y de qué forma falla (formato incorrecto, contenido incorrecto, demasiado largo, etc.).

4

Hacer un único cambio

Cambia una sola variable a la vez. Si cambias varias cosas simultáneamente, no sabrás qué funcionó. Documenta cada versión.

5

Repetir hasta cumplir el criterio

Itera hasta que el prompt pase el conjunto de evaluación. En producción, esto se automatiza con evals (Fase 7.2).

Qué medir

✅ Correctitud

¿La respuesta es factualmente correcta? ¿Hace lo que pediste?

📐 Formato

¿Respeta el formato pedido (JSON, lista, longitud máxima)?

🔄 Consistencia

¿Con diferentes inputs similares, el comportamiento es estable?

💰 Eficiencia

¿Produce el output correcto con el mínimo de tokens posible?

Herramienta: Prompt Playground Anthropic Console, OpenAI Playground y LangSmith te permiten hacer A/B testing de prompts con el mismo conjunto de inputs. Úsalos para no perder versiones anteriores.
