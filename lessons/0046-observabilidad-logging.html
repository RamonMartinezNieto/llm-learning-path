7.3 — Observabilidad, logging y monitoreo de aplicaciones IA

Lección 46 de 55 · ~14 min · Fase 7: Sistemas de IA en Producción

Una aplicación LLM en producción sin observabilidad es una caja negra. Cuando algo falla — y fallará — sin logs adecuados es imposible entender por qué. La observabilidad de aplicaciones IA tiene sus propias peculiaridades respecto a aplicaciones tradicionales: los "errores" no siempre son excepciones, a veces son respuestas incorrectas silenciosas.

Qué loguear en cada llamada al LLM

CampoPor qué es importante
Input prompt (system + user)Reproducir el comportamiento exacto para depuración
Output completoDetectar respuestas incorrectas o fuera de formato
Modelo usadoComparar comportamiento entre versiones de modelo
Tokens de entrada / salidaMonitorizar coste y detectar prompts que crecen
Latencia (ms)Detectar degradación de rendimiento del proveedor
Coste ($)Monitorizar gasto por feature/usuario
User ID / Session IDCorrelacionar problemas con usuarios específicos
TimestampDetectar patrones temporales de fallos

Métricas clave a monitorear

  • Error rate: % de llamadas que devuelven error (timeout, rate limit, malformed output).
  • Coste por tarea: Tokens totales usados para completar una tarea de negocio end-to-end.
  • Latencia p50/p99: La mediana y el percentil 99 de tiempo de respuesta.
  • Tasa de reintentos: Cuántas veces el sistema necesita reintentar antes de éxito.
  • Hallucination rate: Medido mediante evals periódicas sobre muestras de producción.

Herramientas de observabilidad LLM

Langfuse: Open source, auto-hosteable. Registra trazas, métricas de tokens, anotaciones y evals. Muy recomendado para equipos que necesitan control total sobre sus datos.

Helicone: Proxy transparente entre tu aplicación y la API del LLM. Sin cambios de código, captura todas las llamadas automáticamente.

Datadog LLM Observability: Para equipos que ya usan Datadog — integra métricas LLM con el resto del stack de infraestructura.

Alertas: cuándo notificar

Configura alertas para:

  • Error rate > 5% en ventana de 5 minutos
  • Latencia p99 > umbral definido por SLA
  • Gasto diario de tokens supera el presupuesto
  • Tasa de respuestas con formato incorrecto (detectable con evals de regex)

Privacidad: el elefante en la habitación

Los prompts de usuario pueden contener datos personales sensibles — nombres, emails, información de salud, datos financieros. Antes de loguear prompts completos:

  • ¿Tu política de privacidad permite almacenar estas interacciones?
  • ¿Cumples con GDPR/LOPD respecto al derecho de supresión?
  • ¿Tienes acceso control sobre quién puede ver los logs de prompts?
  • ¿Aplicas anonimización/pseudonimización antes de almacenar?
Equilibrio crítico Loguea suficiente para depurar cualquier fallo, pero no más. Los prompts de usuarios pueden contener PII — asegúrate de cumplir con GDPR/LOPD antes de almacenarlos.
