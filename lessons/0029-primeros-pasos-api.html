5.1 — Primeros pasos con la API de Anthropic / OpenAI

Fase 5 · Lección 29 de 55 · ~18 min · Construcción con APIs de IA

Las APIs de los grandes modelos de lenguaje te permiten integrar inteligencia artificial en cualquier aplicación. En esta lección aprendemos lo esencial para hacer tu primera llamada real desde Python.

Autenticación y API keys

Para usar las APIs de Anthropic u OpenAI necesitas una clave de API (API key). Esta clave identifica tu cuenta y se usa para facturar el uso. Hay una regla de oro:

Seguridad crítica Las API keys son credenciales de acceso con coste. Guárdalas en variables de entorno (.env) nunca en el código ni en el repositorio. Una key expuesta en GitHub puede generar miles de euros en cargos en pocas horas.

La forma correcta es usar un archivo .env en la raíz de tu proyecto (añadido al .gitignore) y cargarlo con la librería python-dotenv:

# archivo .env (NUNCA subir a git)
ANTHROPIC_API_KEY=sk-ant-api03-...
OPENAI_API_KEY=sk-proj-...
# Python: cargar la key desde .env
from dotenv import load_dotenv
import os

load_dotenv()
api_key = os.environ.get("ANTHROPIC_API_KEY")
# Las librerías la leen automáticamente del entorno

La estructura de mensajes: roles del sistema

Tanto Anthropic como OpenAI usan el mismo concepto: un array de mensajes donde cada mensaje tiene un role y un content. Los roles posibles son:

RoleDescripciónCuándo se usa
systemInstrucciones base del asistenteAl inicio, define comportamiento y personalidad
userMensajes del usuario humanoCada turno del usuario en la conversación
assistantRespuestas anteriores del modeloPara mantener historial de conversación

Llamada básica con la API de Anthropic

Instala la librería con pip install anthropic y realiza tu primera llamada:

import anthropic

client = anthropic.Anthropic()
# La key se lee de ANTHROPIC_API_KEY en el entorno

message = client.messages.create(
    model="claude-opus-4-5",
    max_tokens=1024,
    system="Eres un asistente útil y conciso.",  # opcional
    messages=[
        {"role": "user", "content": "Hola, ¿cómo estás?"}
    ]
)

# Acceder al texto de la respuesta
print(message.content[0].text)

# Información útil en la respuesta
print(f"Tokens de entrada: {message.usage.input_tokens}")
print(f"Tokens de salida: {message.usage.output_tokens}")

Llamada básica con la API de OpenAI

Instala con pip install openai. La estructura es muy similar:

from openai import OpenAI

client = OpenAI()
# La key se lee de OPENAI_API_KEY en el entorno

completion = client.chat.completions.create(
    model="gpt-4o",
    messages=[
        {"role": "system", "content": "Eres un asistente útil y conciso."},
        {"role": "user", "content": "Hola, ¿cómo estás?"}
    ]
)

# Acceder al texto de la respuesta
print(completion.choices[0].message.content)

# Información de uso
print(f"Tokens totales: {completion.usage.total_tokens}")

Campos clave de la respuesta

Las dos APIs tienen estructuras de respuesta ligeramente diferentes. Es importante conocer dónde está el texto:

APICómo acceder al textoTokens de uso
Anthropicmessage.content[0].textmessage.usage.input_tokens
OpenAIcompletion.choices[0].message.contentcompletion.usage.total_tokens

Nota que Anthropic devuelve content como un array (puede haber múltiples bloques, por ejemplo texto + tool use), mientras que OpenAI devuelve choices como array (para cuando se usan múltiples completions con n>1).

Conversación multi-turno

Para mantener una conversación, simplemente acumulas los mensajes en el array:

import anthropic

client = anthropic.Anthropic()
historial = []

def chatear(mensaje_usuario):
    historial.append({"role": "user", "content": mensaje_usuario})
    
    respuesta = client.messages.create(
        model="claude-opus-4-5",
        max_tokens=1024,
        messages=historial
    )
    
    texto = respuesta.content[0].text
    historial.append({"role": "assistant", "content": texto})
    return texto

print(chatear("Me llamo Ana."))
print(chatear("¿Cómo me llamo?"))  # El modelo sabrá que te llamas Ana
