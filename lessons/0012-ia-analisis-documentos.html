IA para análisis de documentos y datos

Fase 2 · Lección 12 de 55 · ~13 min · Chat Mode Avanzado

Qué puedes hacer con un documento en el contexto

Pegando o adjuntando un documento al contexto de un LLM puedes realizar tareas que antes requerían horas de lectura y análisis manual. La clave es saber qué pedir.

Extracción: "Lista todas las fechas y compromisos mencionados en este contrato."
Resumen ejecutivo: "Resume este informe en 5 bullets para una audiencia no técnica."
Comparación: "Compara estos dos documentos e identifica las diferencias clave en los términos de pago."
Q&A: "Basándote solo en este documento, ¿qué dice sobre los plazos de devolución?"
Detección de problemas: "¿Hay inconsistencias o cláusulas ambiguas en este documento que deberían revisarse?"
Limitación crítica: grounding El modelo puede generar información que parece estar en el documento pero no está. Para análisis críticos (contratos, documentos legales, médicos), siempre verifica manualmente los puntos clave. Pide siempre que el modelo cite el párrafo exacto del que extrae la información.

Plantilla para análisis de documento

Prompt de análisis con grounding Analiza el siguiente documento. Para cada punto que identifiques:
- Cita textualmente el fragmento del que lo extraes (entre comillas)
- Indica la sección/página si está disponible
- Si no encuentras información sobre X en el documento, dilo explícitamente

Tarea: [Tu análisis específico]

[DOCUMENTO]
{pega el documento aquí}
Para datos estructurados (CSV, tablas) Los LLMs con code interpreter (ChatGPT, Claude con artefactos) pueden ejecutar código Python para analizar CSVs con pandas. Esto da análisis cuantitativo verificable, no solo interpretación textual.
