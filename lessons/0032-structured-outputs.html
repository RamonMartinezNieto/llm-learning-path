5.4 — Structured outputs y parsing de respuestas

Fase 5 · Lección 32 de 55 · ~14 min · Construcción con APIs de IA

Cuando el output del LLM va a ser procesado por código —almacenado en una base de datos, pasado a otro sistema, mostrado en una UI estructurada— necesitas que la respuesta tenga un formato predecible. Los structured outputs te dan exactamente eso.

¿Por qué importan los structured outputs?

Sin structured outputs, dependes de que el modelo respete voluntariamente un formato. Puede que funcione el 95% del tiempo, pero ese 5% de fallos rompe tu pipeline. Los enfoques de structured output garantizan la estructura de la respuesta, eliminando la necesidad de regex frágiles o parsing manual.

Método 1: Instrucción en el prompt

El más sencillo: pedirle explícitamente al modelo que responda en JSON con un schema específico. Funciona bien para casos de uso sencillos:

import anthropic
import json

client = anthropic.Anthropic()

texto_a_analizar = "El producto llegó roto y el servicio de atención fue horrible."

respuesta = client.messages.create(
    model="claude-opus-4-5",
    max_tokens=512,
    system="""Eres un analizador de sentimientos. 
Responde SIEMPRE en JSON válido con exactamente este schema:
{
  "sentimiento": "positivo" | "neutro" | "negativo",
  "puntuacion": número entre -1.0 y 1.0,
  "aspectos": [lista de aspectos mencionados],
  "resumen": "frase breve"
}
No incluyas ningún texto fuera del JSON.""",
    messages=[{"role": "user", "content": texto_a_analizar}]
)

# Parsear el JSON
datos = json.loads(respuesta.content[0].text)
print(datos["sentimiento"])   # → "negativo"
print(datos["puntuacion"])    # → -0.85

Método 2: OpenAI con response_format

OpenAI ofrece un modo de structured output nativo que garantiza conformidad con el schema JSON:

from openai import OpenAI
import json

client = OpenAI()

respuesta = client.chat.completions.create(
    model="gpt-4o",
    response_format={"type": "json_schema", "json_schema": {
        "name": "analisis_sentimiento",
        "strict": True,
        "schema": {
            "type": "object",
            "properties": {
                "sentimiento": {"type": "string", "enum": ["positivo", "neutro", "negativo"]},
                "puntuacion": {"type": "number"},
                "resumen": {"type": "string"}
            },
            "required": ["sentimiento", "puntuacion", "resumen"],
            "additionalProperties": False
        }
    }},
    messages=[
        {"role": "system", "content": "Analiza el sentimiento del texto."},
        {"role": "user", "content": "El producto llegó roto y el servicio fue horrible."}
    ]
)

datos = json.loads(respuesta.choices[0].message.content)
print(datos)  # Garantizado que sigue el schema exacto

Método 3: Anthropic con validación Pydantic

Con Anthropic puedes combinar instrucciones de prompt con validación Pydantic para tener tipado fuerte en Python:

import anthropic
import json
from pydantic import BaseModel, ValidationError
from typing import Literal, List

client = anthropic.Anthropic()

# Definir el schema con Pydantic
class AnalisisSentimiento(BaseModel):
    sentimiento: Literal["positivo", "neutro", "negativo"]
    puntuacion: float  # Entre -1.0 y 1.0
    aspectos: List[str]
    resumen: str

def analizar_sentimiento(texto: str) -> AnalisisSentimiento:
    respuesta = client.messages.create(
        model="claude-opus-4-5",
        max_tokens=512,
        system=f"""Analiza el sentimiento. Responde SOLO en JSON con este schema:
{AnalisisSentimiento.model_json_schema()}
Sin texto adicional fuera del JSON.""",
        messages=[{"role": "user", "content": texto}]
    )
    
    json_texto = respuesta.content[0].text
    
    try:
        datos = json.loads(json_texto)
        resultado = AnalisisSentimiento(**datos)  # Valida el schema
        return resultado
    except (json.JSONDecodeError, ValidationError) as e:
        raise ValueError(f"El modelo devolvió un formato inválido: {e}")

resultado = analizar_sentimiento("El producto llegó roto.")
print(resultado.sentimiento)   # Tipado, con autocompletado en el IDE
print(resultado.puntuacion)    # Float garantizado
Advertencia importante Un modelo puede devolver JSON válido con campos correctos pero valores alucinados. El schema garantiza la estructura, no la veracidad del contenido. Valida los valores de negocio por separado.

Casos de uso donde es imprescindible

  • Extracción de datos: entidades, fechas, precios desde texto no estructurado
  • Pipelines de clasificación: categorizar tickets, emails, documentos
  • Relleno de formularios: convertir texto libre en campos estructurados
  • APIs que devuelven datos al frontend: necesitas JSON predecible
  • ETL con LLM: transformar y cargar datos en bases de datos
No confundas estructura con veracidad Si extraes la fecha de nacimiento de un CV y el modelo inventa una fecha en formato correcto, el structured output no te protege de eso. Añade validaciones de dominio (rangos, listas permitidas, verificación cruzada) para detectar valores alucinados.
