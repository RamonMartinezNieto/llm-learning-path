5.3 — Tool use y function calling

Fase 5 · Lección 31 de 55 · ~18 min · Construcción con APIs de IA

Hasta ahora los modelos solo podían devolver texto. Con tool use (llamado function calling en OpenAI), el modelo puede solicitar que tu código ejecute funciones reales y usar el resultado para completar su respuesta. Es el primer paso hacia los agentes de IA.

¿Qué es tool use?

Tool use permite que un LLM "llame" a funciones externas durante su proceso de razonamiento. Aclaración crucial: el modelo no ejecuta nada. Lo que hace es devolver una instrucción estructurada con el nombre de la función y los argumentos. Tu código ejecuta la función real y devuelve el resultado al modelo para que continúe.

Concepto clave Tool use convierte un LLM en un agente básico. La descripción de la herramienta es crucial — el modelo decide si usarla basándose solo en tu descripción.

El ciclo completo de tool use

El flujo tiene cuatro pasos que pueden repetirse en múltiples turnos:

1. Tu código → API: "Aquí tienes las herramientas disponibles y la pregunta del usuario"
2. Modelo → Tu código: "Quiero llamar a get_weather con ciudad='Madrid'"  (stop_reason: "tool_use")
3. Tu código → Modelo: "El resultado de get_weather fue: 22°C, soleado"   (role: "tool")
4. Modelo → Tu código: "En Madrid ahora mismo hay 22°C y está soleado."   (respuesta final)

Definir una herramienta en Anthropic

Las herramientas se definen con un nombre, descripción y schema de los parámetros de entrada en formato JSON Schema:

import anthropic
import json

client = anthropic.Anthropic()

# Definición de la herramienta
tools = [
    {
        "name": "get_weather",
        "description": "Obtiene el tiempo meteorológico actual para una ciudad. "
                       "Úsala cuando el usuario pregunte por el clima o temperatura "
                       "de cualquier lugar.",
        "input_schema": {
            "type": "object",
            "properties": {
                "city": {
                    "type": "string",
                    "description": "Nombre de la ciudad, ej: 'Madrid', 'Barcelona'"
                },
                "units": {
                    "type": "string",
                    "enum": ["celsius", "fahrenheit"],
                    "description": "Unidad de temperatura"
                }
            },
            "required": ["city"]
        }
    }
]

# Primera llamada: el modelo puede decidir usar la herramienta
respuesta = client.messages.create(
    model="claude-opus-4-5",
    max_tokens=1024,
    tools=tools,
    messages=[
        {"role": "user", "content": "¿Qué tiempo hace en Madrid ahora mismo?"}
    ]
)

print(f"Stop reason: {respuesta.stop_reason}")
# → "tool_use" si el modelo quiere usar una herramienta

Manejar la respuesta y completar el ciclo

Cuando stop_reason == "tool_use", extraes el bloque tool_use, ejecutas la función real y devuelves el resultado:

import anthropic
import json

client = anthropic.Anthropic()

# Función real que ejecuta tu código
def get_weather(city: str, units: str = "celsius") -> dict:
    # En producción: llamarías a una API meteorológica real
    return {"city": city, "temperature": 22, "condition": "soleado", "units": units}

def llamar_con_tools(mensaje_usuario: str):
    mensajes = [{"role": "user", "content": mensaje_usuario}]
    
    while True:
        respuesta = client.messages.create(
            model="claude-opus-4-5",
            max_tokens=1024,
            tools=tools,  # definidos arriba
            messages=mensajes
        )
        
        if respuesta.stop_reason == "end_turn":
            # El modelo terminó sin usar más herramientas
            return respuesta.content[0].text
        
        if respuesta.stop_reason == "tool_use":
            # Añadir respuesta del modelo al historial
            mensajes.append({"role": "assistant", "content": respuesta.content})
            
            # Procesar cada tool_use en la respuesta
            resultados_tools = []
            for bloque in respuesta.content:
                if bloque.type == "tool_use":
                    # Ejecutar la función real
                    if bloque.name == "get_weather":
                        resultado = get_weather(**bloque.input)
                    
                    resultados_tools.append({
                        "type": "tool_result",
                        "tool_use_id": bloque.id,
                        "content": json.dumps(resultado)
                    })
            
            # Devolver resultados al modelo
            mensajes.append({"role": "user", "content": resultados_tools})

respuesta = llamar_con_tools("¿Qué tiempo hace en Madrid?")
print(respuesta)

Casos de uso habituales

HerramientaQué haceEjemplo de uso
Búsqueda webConsulta información actualNoticias, precios, datos recientes
Base de datosConsulta datos de tu sistemaEstado de un pedido, saldo de cuenta
Ejecución de códigoCálculos, transformacionesAnálisis de datos, gráficas
Calendario / emailAcceso a herramientas de productividadCrear evento, enviar mensaje
APIs externasIntegración con serviciosMapas, pagos, notificaciones
Seguridad importante Valida siempre los argumentos que devuelve el modelo antes de ejecutar la función. El modelo podría generar argumentos inesperados o maliciosos. Nunca ejecutes código arbitrario que el modelo sugiera sin validación.
