El principio de especificidad

El modelo asigna probabilidad a los tokens. Un prompt vago abre el espacio de respuestas posibles. Un prompt específico lo cierra. La especificidad es tu palanca más directa sobre la calidad del output.

La pregunta diagnóstico Antes de enviar un prompt, pregúntate: "Si 10 humanos diferentes leyeran esto, ¿responderían de forma muy similar?" Si la respuesta es no, el prompt necesita más especificidad.

System prompt vs. User prompt

La mayoría de APIs separan system prompt (instrucciones de rol y comportamiento permanente) del user prompt (la tarea concreta). El system prompt persiste como contexto en toda la conversación.
