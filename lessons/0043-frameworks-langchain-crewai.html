6.7 — Frameworks: LangChain, AutoGen, CrewAI, Claude Agent SDK

Lección 43 de 55 · ~14 min · Fase 6: Código Agéntico

Los frameworks de agentes ofrecen abstracciones que aceleran el desarrollo: gestión del bucle, integración de herramientas, manejo de memoria, logging. Sin embargo, cada nivel de abstracción es también un nivel de opacidad. Elegir un framework sin entender los patrones subyacentes es una receta para frustraciones difíciles de depurar.

Los principales frameworks

LangChain: el más maduro

LangChain lleva más tiempo en el mercado y tiene el ecosistema más amplio de integraciones (VectorStores, Document Loaders, Tools). Introduce LCEL (LangChain Expression Language) para componer chains de forma declarativa. Es la mejor opción si necesitas integraciones rápidas con fuentes de datos externas o si tu equipo ya lo conoce.

AutoGen: especialización multi-agent

AutoGen de Microsoft Research está diseñado específicamente para sistemas donde múltiples agentes conversan entre sí para resolver problemas. Soporta agentes humanos y de IA en el mismo loop. Ideal para investigación y prototipado de sistemas conversacionales complejos.

CrewAI: enfoque basado en roles

CrewAI modela el sistema como un equipo (crew) con agentes que tienen roles, goals y backstories. La API es más accesible y el tiempo de prototipado es menor. Pierde flexibilidad para arquitecturas no estándar.

El tradeoff fundamental: abstracción vs. control

Cada framework añade una capa entre tú y el LLM. Esa capa puede:

  • Acelerar el desarrollo con patterns pre-construidos
  • Añadir bugs propios que no entiendes
  • Ocultar los tokens reales que se envían al modelo
  • Deprecarse o cambiar breaking changes entre versiones

La recomendación: aprende los patrones primero (ReAct, tool design, agent loops). Luego usa el framework como acelerador, no como punto de partida.

Advertencia práctica Los frameworks abstraen complejidad pero también ocultan problemas. Si no entiendes el patrón ReAct, no entenderás por qué tu agente de LangChain falla.
