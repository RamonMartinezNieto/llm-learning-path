5.7 — RAG: Retrieval-Augmented Generation

Fase 5 · Lección 35 de 55 · ~18 min · Construcción con APIs de IA

Los LLMs solo conocen lo que estaba en sus datos de entrenamiento. No conocen los documentos internos de tu empresa, la base de conocimiento de tu producto, ni nada publicado después de su fecha de corte. RAG es el patrón que resuelve exactamente este problema.

El problema que resuelve RAG

Supón que quieres un chatbot que responda preguntas sobre el manual interno de tu empresa. Opciones sin RAG:

  • Incluir todo el manual en el prompt: prohibitivo en coste si el manual tiene miles de páginas
  • Fine-tuning: costoso, lento, y el modelo "memoriza" el conocimiento en lugar de citarlo
  • Confiar en el modelo base: no conoce tu manual, simplemente alucinará

RAG ofrece una cuarta opción: en tiempo de consulta, recuperar solo los fragmentos del manual relevantes para la pregunta y pasarlos en el prompt. Eficiente, actualizable y con trazabilidad.

Estadística del sector RAG es el patrón más importante para conectar LLMs con datos privados de empresas. El 80% de los casos de uso empresarial de IA generativa son RAG o variantes de RAG.

El pipeline RAG: paso a paso

1
Ingesta de documentos: Cargas tus documentos (PDFs, páginas web, bases de datos, Notion, etc.) en el sistema.
2
Chunking: Divides cada documento en fragmentos (chunks) de tamaño manejable. La estrategia importa: chunks demasiado pequeños pierden contexto; demasiado grandes diluyen la relevancia.
3
Embedding: Conviertes cada chunk en un vector numérico usando un modelo de embeddings. Chunks con significado similar tendrán vectores similares.
4
Almacenamiento: Guardas los vectores en una base de datos vectorial (Chroma, Pinecone, pgvector...).
5
Consulta (en runtime): Cuando llega una pregunta del usuario, la conviertes también en un vector con el mismo modelo de embeddings.
6
Recuperación: Buscas en la DB vectorial los chunks más similares a la pregunta (por similitud coseno).
7
Inyección en prompt: Incluyes los chunks recuperados en el prompt como contexto para el LLM.
8
Generación: El LLM responde la pregunta basándose en el contexto recuperado.

Implementación básica con Chroma y Anthropic

import anthropic
from openai import OpenAI
import chromadb

# Clientes
anthropic_client = anthropic.Anthropic()
openai_client = OpenAI()
chroma_client = chromadb.Client()
coleccion = chroma_client.create_collection("documentos_empresa")

# FASE 1: INDEXACIÓN (se hace una vez, o cuando cambian los docs)
documentos = [
    "La política de vacaciones permite 22 días laborables al año.",
    "Las solicitudes de vacaciones se hacen con 15 días de antelación mínima.",
    "El horario de trabajo es flexible entre las 8:00 y las 18:00.",
    "El teletrabajo está permitido hasta 3 días por semana.",
    "Las bajas por enfermedad requieren justificante médico a partir del 3er día.",
]

# Generar embeddings para cada documento
embeddings = []
for doc in documentos:
    respuesta = openai_client.embeddings.create(
        model="text-embedding-3-small",
        input=doc
    )
    embeddings.append(respuesta.data[0].embedding)

# Almacenar en Chroma
coleccion.add(
    documents=documentos,
    embeddings=embeddings,
    ids=[f"doc_{i}" for i in range(len(documentos))]
)

# FASE 2: CONSULTA (se hace en cada pregunta del usuario)
def preguntar(pregunta: str) -> str:
    # 1. Convertir pregunta a embedding
    embedding_pregunta = openai_client.embeddings.create(
        model="text-embedding-3-small",
        input=pregunta
    ).data[0].embedding
    
    # 2. Recuperar chunks más relevantes
    resultados = coleccion.query(
        query_embeddings=[embedding_pregunta],
        n_results=3
    )
    
    chunks_relevantes = resultados["documents"][0]
    contexto = "\n".join(f"- {chunk}" for chunk in chunks_relevantes)
    
    # 3. Llamar al LLM con el contexto
    respuesta = anthropic_client.messages.create(
        model="claude-opus-4-5",
        max_tokens=512,
        system=f"""Eres un asistente de RRHH. Responde basándote SOLO en el siguiente 
contexto. Si la respuesta no está en el contexto, dilo explícitamente.

Contexto relevante:
{contexto}""",
        messages=[{"role": "user", "content": pregunta}]
    )
    
    return respuesta.content[0].text

# Probar
print(preguntar("¿Cuántos días de vacaciones tengo?"))
print(preguntar("¿Puedo trabajar desde casa todos los días?"))

Estrategias de chunking

EstrategiaCómo funcionaMejor para
Tamaño fijoCada N caracteres o tokensDocumentos sin estructura clara
Por párrafosDividir por saltos de línea doblesArtículos, manuales
Por seccionesDividir por headings (H1, H2...)Documentación técnica
SemánticoDetectar cambios de temaDocumentos heterogéneos
Con solapamientoChunks con overlap del 10-20%Cuando el contexto entre chunks importa

RAGFine-tuning
Datos que cambian✅ Ideal❌ Requiere reentrenar
