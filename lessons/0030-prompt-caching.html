5.2 — Prompt caching y optimización de costes

Fase 5 · Lección 30 de 55 · ~13 min · Construcción con APIs de IA

Cada token que envías a la API tiene un coste. Cuando tu aplicación repite los mismos bloques de texto en cada llamada —un system prompt largo, ejemplos few-shot, documentos de referencia— estás pagando por procesarlos repetidamente. El prompt caching elimina ese desperdicio.

¿Qué es el prompt caching?

El prompt caching permite al servidor de Anthropic guardar en caché el resultado de procesar un prefijo de prompt. En llamadas posteriores que compartan ese mismo prefijo, el servidor reutiliza el procesamiento ya hecho. Tú solo pagas una fracción del coste normal por esos tokens en caché.

Es análogo al caché de una CPU: la primera vez hay que calcular, pero las siguientes veces se lee directamente del caché a mucho menor coste.

¿Cuándo es efectivo?

El caching tiene mayor impacto en estos escenarios:

  • System prompts extensos con instrucciones detalladas (>1024 tokens)
  • Ejemplos few-shot numerosos al inicio del prompt
  • Documentos de referencia o bases de conocimiento inyectados en el contexto
  • Plantillas de análisis repetidas con distintos inputs variables
Regla práctica El caching es más efectivo cuando el prefijo del prompt tiene >1024 tokens y se reutiliza en muchas llamadas. Un system prompt largo + RAG docs = candidato perfecto para caching.

Cómo habilitarlo en la API de Anthropic

Se añade el campo cache_control con {"type": "ephemeral"} al mensaje que quieres cachear. Anthropic mantiene la caché por 5 minutos (renovable con cada acceso):

import anthropic

client = anthropic.Anthropic()

# System prompt largo que se repite en cada llamada
SYSTEM_PROMPT_LARGO = """
Eres un asistente especializado en análisis legal.
[...cientos de líneas de instrucciones...]
""" * 50  # Simulando un prompt largo

respuesta = client.messages.create(
    model="claude-sonnet-4-5",
    max_tokens=1024,
    system=[
        {
            "type": "text",
            "text": SYSTEM_PROMPT_LARGO,
            "cache_control": {"type": "ephemeral"}  # ← Activa el caching
        }
    ],
    messages=[
        {"role": "user", "content": "Analiza este contrato: ..."}
    ]
)

# Verificar si se usó caché
uso = respuesta.usage
print(f"Tokens de entrada: {uso.input_tokens}")
print(f"Tokens leídos de caché: {uso.cache_read_input_tokens}")
print(f"Tokens escritos en caché: {uso.cache_creation_input_tokens}")

Impacto en costes: comparativa de precios

Para entender el ahorro, veamos la estructura de precios de Claude Sonnet (valores ilustrativos, consulta la documentación oficial para precios actualizados):

Tipo de tokenCoste relativoDescripción
Input normal100%Tokens procesados por primera vez, sin caché
Cache write~125%Primera vez que se guarda en caché (pequeño sobrecargo)
Cache read~10%Tokens leídos del caché (ahorro ~90%)
Output~500%Tokens generados (siempre los más caros)

Ejemplo concreto: si tienes un system prompt de 5.000 tokens y haces 100 llamadas al día, sin caching pagas 500.000 tokens de input. Con caching pagas ~6.250 tokens (1 escritura + 99 lecturas × 5.000 × 10% ≈ 49.625 tokens). Ahorro aproximado: 90% en esos tokens.

Caching de documentos de referencia (RAG)

Un patrón muy potente es cachear documentos extensos que se consultan repetidamente:

import anthropic

client = anthropic.Anthropic()

# Documento de referencia extenso (manual, base de conocimiento, etc.)
documento_referencia = open("manual_empresa.txt").read()

def analizar_con_contexto(pregunta: str):
    return client.messages.create(
        model="claude-sonnet-4-5",
        max_tokens=512,
        system=[
            {
                "type": "text",
                "text": "Eres un asistente de soporte. Responde basándote en el manual.",
                "cache_control": {"type": "ephemeral"}
            },
            {
                "type": "text",
                "text": f"Manual de la empresa:\n\n{documento_referencia}",
                "cache_control": {"type": "ephemeral"}  # ← Cachea el documento
            }
        ],
        messages=[{"role": "user", "content": pregunta}]
    )

# Primera llamada: escribe en caché (un poco más cara)
r1 = analizar_con_contexto("¿Cuál es la política de vacaciones?")

# Segunda llamada: lee del caché (90% más barata en esos tokens)
r2 = analizar_con_contexto("¿Cómo solicito días de enfermedad?")
Atención El caché de Anthropic expira tras 5 minutos de inactividad. Si tus llamadas están muy espaciadas en el tiempo o en producción hay baja carga, puede que el beneficio sea menor de lo esperado. Monitoriza los campos cache_read_input_tokens en las respuestas para verificar que el caching está funcionando.
