Ejemplo de workflow: clasificación automática de emails

Un workflow real que puedes construir en cualquiera de las tres plataformas:

Trigger: Nuevo email en Gmail

→ Nodo IA: Clasificar email con LLM
  Prompt: "Clasifica este email en una categoría:
  URGENTE, CLIENTE, NEWSLETTER, SPAM, INTERNO.
  Solo responde con la categoría.
  Email: {email_body}"

→ Router: Según categoría
  - URGENTE → Slack #alertas + Gmail etiqueta "URGENTE"
  - CLIENTE  → HubSpot crear tarea + Gmail carpeta "Clientes"
  - NEWSLETTER → Gmail carpeta "Para leer después"
  - SPAM     → Gmail mover a spam
  - INTERNO  → Gmail etiqueta "Interno"

Cuándo elegir n8n sobre Zapier/Make

n8n brilla especialmente cuando:

  • Los datos no pueden salir de tu infraestructura (GDPR, datos médicos, secretos empresariales)
  • Volumen de ejecuciones alto que haría prohibitivo el precio de Zapier/Make
  • Necesitas integrar modelos de lenguaje locales (Ollama, modelos self-hosted)
  • Quieres lógica de negocio compleja con código JavaScript personalizado
  • El flujo tiene ramas condicionales complejas que las otras plataformas no soportan bien

Próximos pasos tras completar el plan

Construye algo real: El mejor aprendizaje viene de enfrentarse a problemas reales con deadline real. Elige un proyecto pequeño pero completo: un chatbot para tu empresa, un pipeline de análisis de datos, un agente que automatice una tarea repetitiva tuya.

Comunidad: Únete a comunidades de practitioners: Latent Space (podcast + Discord), Towards Data Science, LangChain Discord, Hugging Face forums. El campo evoluciona cada semana — la comunidad es tu fuente más rápida de actualización.

Contribución open source: Contribuir a proyectos de IA open source (LangChain, n8n, Langfuse, llama.cpp) es la forma más efectiva de aprender en profundidad y construir reputación en el campo.
