6.6 — Evaluación y trazabilidad de agentes

Lección 42 de 55 · ~14 min · Fase 6: Código Agéntico

Evaluar un agente es fundamentalmente más difícil que evaluar un prompt simple. Los agentes son no-deterministas, multi-paso, y sus fallos pueden emerger de combinaciones de acciones que son imposibles de anticipar mirando cada paso individualmente. Sin embargo, la evaluación sistemática y la trazabilidad son imprescindibles para operar agentes en producción.

Por qué es difícil evaluar agentes

  • No-determinismo: El mismo agente con el mismo input puede tomar caminos diferentes en ejecuciones distintas. Las métricas necesitan ser estadísticas, no puntuales.
  • Multi-paso: El fallo puede ocurrir en el paso 1 (planning incorrecto), en el paso 7 (error de herramienta mal manejado) o en el paso 12 (pérdida de coherencia del plan). Es difícil asignar la causa.
  • Fallos emergentes: Un agente puede completar correctamente cada paso individual pero producir un resultado global incorrecto por la composición de decisiones subóptimas.

Herramientas de trazabilidad

LangSmith (LangChain): Registra cada llamada al LLM, cada tool call y cada observación. Permite visualizar el árbol de decisiones del agente y comparar ejecuciones. Tiene soporte nativo para evals sobre las trazas.

Langfuse: Open source, auto-hosteable. Registra spans jerárquicos (la traza completa > cada iteración > cada llamada). Ideal para equipos que necesitan control total sobre sus datos de observabilidad.

Anthropic Console: Para agentes basados en Claude, el Console muestra las trazas de tool use directamente sin necesidad de integración adicional.

Cómo construir un test suite para agentes

Un buen test suite para agentes incluye:

  1. Casos de éxito normales: Tareas típicas que el agente debería completar sin problemas.
  2. Casos límite: Inputs ambiguos, herramientas que fallan, contextos incompletos.
  3. Tests de regresión: Tareas que fallaron en el pasado y ahora deben pasar.
  4. Tests de coste: Verificar que tareas simples no usan más de N iteraciones/tokens.

Seeds reproducibles para depuración

Para depurar un fallo específico, necesitas poder reproducirlo. Configura siempre una seed de temperatura (temperatura = 0 + seed fija) en el test suite para hacer las ejecuciones deterministas. En producción puedes usar temperatura > 0, pero en tests la reproducibilidad es prioritaria.

Principio de producción La trazabilidad no es opcional en producción. Sin logs detallados de cada paso del agente, depurar fallos es casi imposible.
